# Phase 8 — المتطلبات غير الوظيفية

**الهدف:** سجل النشاط، النسخ الاحتياطي، حماية الملفات، الطباعة، الأداء، التحسين النهائي.

**يعتمد على:** كل المراحل (طبقة عرضية تُطبَّق تدريجيًا وتُكتمل هنا).

---

## المهام

### سجل النشاط (Audit)
- [ ] Migration `audit_logs` + Trait `Auditable` (created/updated/deleted) + تسجيل الدخول.
- [ ] شاشة عرض السجل (لرئيس التوجيه) + فلترة.

### النسخ الاحتياطي
- [ ] `spatie/laravel-backup` أو أمر Artisan مجدول + شاشة إدارة/تنزيل.

### حماية الملفات
- [ ] تخزين خاص (`storage/app/private`) + مسار توصيل محمي بصلاحية (`FileController` + Policy).

### الإشعارات
- [ ] جدول `notifications` + إشعارات داخل التطبيق (زيارة متأخرة، مهمة جديدة، اعتماد...).
- [ ] جرس الإشعارات (wiggle عند غير مقروء).

### الطباعة
- [ ] إكمال `PrintLayout` (A4، ترويسة/تذييل، «صفحة X من Y»، إزالة الظلال).

### الأداء والتوسع
- [ ] فهارس DB (انظر [`02-DATABASE.md`](../02-DATABASE.md) §4) + eager loading + caching للمؤشرات.
- [ ] Jobs للعمليات الثقيلة (استيراد/تقارير).

### الجودة النهائية
- [ ] تدقيق RBAC شامل + اختبارات.
- [ ] تجاوب الجوّال (Bottom Nav/Sheet) عبر كل الصفحات.
- [ ] `prefers-reduced-motion` + ARIA + حالات التركيز.
- [ ] `npm run build` نظيف + مراجعة كود.

---

## معايير القبول
- كل عملية حسّاسة مسجّلة في `audit_logs`.
- النسخ الاحتياطي يعمل ويمكن استرجاعه.
- لا وصول مباشر للملفات بلا صلاحية.
- الطباعة نظيفة، الواجهة متجاوبة، الأداء جيد.

---

## المرحلة الثانية (تطوير مستقبلي — خارج النطاق الحالي)
- تطبيق جوّال للموجه.
- توقيع إلكتروني متقدّم.
- إشعارات فورية (push/realtime).
- خرائط المدارس.
- ذكاء اصطناعي للتوصيات + توقّع المدارس عالية الاحتياج للدعم.
- لوحة زمنية (Timeline) للفصل + تجميد العام.
