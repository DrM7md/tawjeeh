# Phase 5 — الزيارات والمتابعة + الاستمارات

**الهدف:** تسجيل الزيارات (معلم/منسق) حسب القواعد + استمارات + توقيع + مرفقات + لوحة متابعة.

**يعتمد على:** Phase 4 (معلمون/منسقون) + Phase 2 (العام/الفصل).

---

## المهام

### قاعدة البيانات
- [ ] Migrations: `visits`, `visit_forms`, `visit_files`. (انظر [`02-DATABASE.md`](../02-DATABASE.md))

### المنطق (قواعد VS / VF في [`05-BUSINESS-RULES.md`](../05-BUSINESS-RULES.md))
- [ ] `VisitService`:
  - زيارة معلم: حد أدنى 1 + إضافية حسب `classification.required_visits` (VS-2/3).
  - زيارة منسق: 1 سنويًا، تُسجَّل بفصلها (VS-4).
  - حساب الحالة: تمت ✓ / متبقي ⏳ / متأخر ⚠️ (VS-5).
  - فصل الإنجاز بين الفصلين (VS-6).
- [ ] `VisitFormService`: مسودة/اعتماد نهائي (VF-2) + قفل بعد الاعتماد (VF-3).
- [ ] Policy: الموجه يسجّل لمدارسه فقط.
- [ ] رفع المرفقات لتخزين خاص محمي.

### الواجهة
- [ ] **لوحة المتابعة**: StatCards (تمت/متبقي/متأخر/نسبة الإنجاز) + جداول قابلة للفلترة (عام/فصل/قسم/موجه/مدرسة/مستهدف).
- [ ] قائمة الزيارات + إنشاء زيارة (Modal): اختيار المدرسة/المستهدف/التاريخ.
- [ ] **استمارة الزيارة**: محاور التقييم، ملاحظات، توصيات، **توقيع إلكتروني**، رفع مرفقات (Dropzone).
- [ ] حفظ كمسودة / اعتماد نهائي + عرض الحالة لكل معلم/منسق.
- [ ] طباعة الاستمارة (PrintLayout).

---

## معايير القبول
- حالات الزيارة تُحسب صحيحًا مقابل التصنيف والمواعيد.
- المنسق لا يظهر كمطلوب في الفصل الآخر بعد زيارته.
- الاستمارة تُحفظ مسودة وتُعتمد وتُقفل.
- المرفقات محمية (لا وصول مباشر بلا صلاحية).
- لوحة المتابعة دقيقة وتفصل بين الفصلين.
