# Phase 3 — توزيع المدارس على الموجهين

**الهدف:** توزيع المدارس بعدالة (تلقائي/يدوي/إعادة) + مؤشرات الحمل.

**يعتمد على:** Phase 1 (مدارس/مستخدمون) + Phase 2 (السياق).

---

## المهام

### قاعدة البيانات
- [ ] Migration: `school_assignments` (انظر [`02-DATABASE.md`](../02-DATABASE.md)) + UNIQUE(year, school, department).

### المنطق
- [ ] `DistributeSchoolsAction` (خوارزمية least-loaded — انظر DS في [`05-BUSINESS-RULES.md`](../05-BUSINESS-RULES.md)):
  - حساب وزن المدرسة = دالة(معلمين، منسقين، زيارات مطلوبة، مرحلة).
  - توزيع تلقائي + معاينة قبل الحفظ.
- [ ] `DistributionService`: توزيع يدوي، إعادة توزيع، نقل مدرسة بين موجهين.
- [ ] حساب مؤشرات: عدد المدارس/موجه، متوسط المعلمين، **نسبة العدالة** (انحراف معياري عن المتوسط).

### الواجهة
- [ ] شاشة التوزيع: عرض الموجهين + حملهم (StatCards) + جدول المدارس.
- [ ] **سحب وإفلات** (drag-drop) لنقل مدرسة بين موجهين.
- [ ] زر «توزيع تلقائي» مع معاينة قابلة للتعديل قبل التأكيد.
- [ ] لوحة الحمل الإشرافي + مؤشر العدالة.
- [ ] فلترة حسب القسم (رئيس القسم يوزّع قسمه فقط — DS-4).

---

## معايير القبول
- توزيع تلقائي ينتج توزيعًا متوازنًا (نسبة عدالة معقولة).
- إعادة التوزيع والسحب/الإفلات يعملان ويُحدّثان المؤشرات لحظيًا.
- لا يمكن إسناد نفس المدرسة/القسم لموجهين في نفس العام.
- النطاق محترم (رئيس القسم لا يوزّع خارج قسمه).
