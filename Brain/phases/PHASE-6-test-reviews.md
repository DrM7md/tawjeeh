# Phase 6 — تحكيم الاختبارات

**الهدف:** تسجيل تحكيم الاختبارات حسب المرحلة والصف والفصل + معايير + أرشفة.

**يعتمد على:** Phase 1 (مدارس/أقسام/مراحل) + Phase 2 (العام/الفصل) + Phase 5 (سياق الزيارة).

---

## المهام

### قاعدة البيانات
- [ ] Migrations: `test_reviews`, `test_review_forms`. (انظر [`02-DATABASE.md`](../02-DATABASE.md))

### المنطق (قواعد TR في [`05-BUSINESS-RULES.md`](../05-BUSINESS-RULES.md))
- [ ] `TestReviewService`:
  - دعم: إعدادي (السابع/الثامن/التاسع)، ثانوي، مشتركة (ابتدائي/إعدادي/ثانوي) (TR-2).
  - ربط بالعام والفصل + تحكيم كل فصل مستقل (TR-1/TR-4).
  - تقييم حسب معايير (`criteria` JSON) + حساب `total_score`.
  - مسودة/اعتماد + أرشفة النتائج (TR-3).
- [ ] رفع نماذج التحكيم (مرفقات).

### الواجهة
- [ ] قائمة سجلات التحكيم + فلترة (عام/فصل/قسم/مدرسة/مرحلة/صف).
- [ ] إنشاء سجل تحكيم (Modal): مدرسة، مرحلة، صف، معايير، درجات.
- [ ] رفع النماذج + أرشفة النتائج.
- [ ] طباعة نتيجة التحكيم.

---

## معايير القبول
- إنشاء تحكيم لكل صف/مرحلة ضمن الفصل الصحيح.
- تحكيم الفصلين مستقل (سجلات منفصلة).
- المعايير والدرجات تُحفظ وتُحسب صحيحًا.
- الأرشفة والطباعة تعملان.
