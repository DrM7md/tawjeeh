# Phase 2 — الأعوام والفصول الدراسية + السياق الفعّال

**الهدف:** بناء نواة السياق (Active Context) التي ترتبط بها كل العمليات.

**يعتمد على:** Phase 1. **يمكّن:** Phases 3–7 (كلها تتعلّق بالسياق).

> ⚠️ مرحلة محورية — يجب إتقانها قبل أي وحدة تشغيلية.

---

## المهام

### قاعدة البيانات
- [ ] Migrations: `academic_years`, `semesters`. (انظر [`02-DATABASE.md`](../02-DATABASE.md))
- [ ] Seeder: عام 2026–2027 نشط + فصلان (الأول نشط).

### منطق السياق
- [ ] Trait `BelongsToAcademicContext` + Global Scope `AcademicContextScope` (فلترة تلقائية + ملء تلقائي عند الإنشاء).
- [ ] Middleware `SetActiveContext` (يقرأ من الجلسة/الافتراضي) + مشاركة عبر Inertia.
- [ ] `AcademicYearService` / `SemesterService`:
  - تفعيل عام واحد فقط (transaction) — AY-1.
  - فصل واحد نشط داخل العام — SM-2.
  - إغلاق/أرشفة — AY-3, SM-5.
  - **نسخ التوزيع** من عام سابق — AY-4.

### الواجهة
- [ ] شاشة إدارة الأعوام (اسم/تواريخ/حالة + إنشاء/تفعيل/إغلاق/أرشفة/نسخ).
- [ ] شاشة إدارة الفصول (مرتبطة بالعام).
- [ ] **محدّد العام/الفصل** في الترويسة (Switcher) — عند التبديل: Inertia reload لكل العدادات/الجداول.
- [ ] منع التعديل عند الإغلاق (read-only mode) في الواجهة + الـ Backend.

---

## قواعد العمل المطبّقة
AY-1 → AY-6، SM-1 → SM-5 (انظر [`05-BUSINESS-RULES.md`](../05-BUSINESS-RULES.md)).

## معايير القبول
- لا يمكن تفعيل عامين أو فصلين معًا (يُختبر).
- إنشاء سجل تجريبي يربطه تلقائيًا بالعام/الفصل الفعّال.
- تبديل السياق من الترويسة يغيّر البيانات المعروضة.
- نسخ توزيع عام سابق يعمل.
